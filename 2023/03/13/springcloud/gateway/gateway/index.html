1 为什么需要网关

（1）如果没有网关，各个微服务就需要对外暴露很多端口，访问不方便并且不安全

（2）如果没有网关，很多微服务都需要单独认证授权，比较麻烦

（3）有些微服务使用的是不同的通信协议，如果没有网关，则客户端访问变得很不方便

等等。。。

5 Glossary[术语]

https://docs.spring.io/spring-cloud-gateway/docs/current/reference/html/#glossary

5.1 Route

The basic building block of the gateway. It is defined by an ID, a destination URI, a collection of predicates, and a collection of filters. A route is matched if the aggregate predicate is true.

5.2 Predicate

This is a Java 8 Function Predicate. The input type is a Spring Framework ServerWebExchange. This lets you match on anything from the HTTP request, such as headers or parameters.

5.3 Filter

These are instances of GatewayFilter that have been constructed with a specific factory. Here, you can modify requests and responses before or after sending the downstream request.

8.2 Nacos Gateway动态刷新原理

（1）根据之前Nacos配置中心源码，可以知道在Nacos配置修改之后，会push数据给client 端

（2）然后在nacos客户端会调用NacosContextRefresher#registerNacosListener

（3）在这段代码中发布了RefreshEvent事件，该事件由RefreshEventListener监听器监听到了

（4）调用this#refresh#refresh()方法

（5）调用this#scope#refreshAll()方法

（6）发布RefreshScopeRefreshEvent事件：this#context#publishEvent(new RefreshScopeRefreshedEvent())

（7）由RouteRefreshListener监听器监听到了，判断事件类型

（8）调用reset方法，发布RefreshRoutesEvent事件

（9）最终调用CachingRouteLocator#onApplicationEvent完成路由规则的更新
